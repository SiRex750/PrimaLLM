# Charon — Semantic Graph Compression Layer

The Charon pipeline compresses raw documents into token-efficient, graph-ranked prose for passage into the HADES memory hierarchy.

## What Charon Does

- Extracts Subject-Verb-Object triples from source text using spaCy dependency parsing.
- Builds a directed NetworkX knowledge graph over entities and relations.
- Ranks triples using PageRank centrality (sum of subject score and object score).
- Maintains an L1 set-associative cache with five typed partitions: SYSTEM, FACTS, HISTORY, TOOLS, and SCRATCH.
- Generates compressed "Charon-style" prose via a local Ollama Small Language Model (qwen2.5:1.5b).

## The SCRATCH Set

The SCRATCH set implements a "dirty bit" write-back policy inspired by computer architecture:

- All LLM-generated content enters the SCRATCH partition first during the inference cycle.
- Content in the SCRATCH partition is tagged as "dirty" (unverified) by default.
- The `flush_scratch()` method evicts all dirty entries and passes them to the Cerberus component for verification.
- Only Cerberus-verified triples graduate from the SCRATCH set to L3 persistent storage.
- This ensures that hallucinations or unverified claims generated by the LLM are filtered before they can contaminate long-term memory.

## SDpT Metric

Semantic Density per Token (SDpT) is the primary metric for measuring compression efficiency in the HADES architecture:

- **SDpT** = `compressed_token_count` / `unique_ACUs_preserved`
- **Lower SDpT** = better compression efficiency (fewer tokens required per Atomic Content Unit).
- **Baseline SDpT** = `raw_token_count` / `total_triples_extracted`
- **Improvement** = `baseline_SDpT` - `charon_SDpT`. A positive value indicates that Charon has improved the information density of the context.

## Module Reference

- **caveman/core/graph.py**: Implements `build_graph()`, `pagerank_scores()`, and `rank_triples_by_importance()`.
- **caveman/core/cache.py**: Manages the L1Cache with 5 typed sets, providing token budget enforcement via `tiktoken`, PageRank-based eviction for the FACTS set, and LRU eviction for the HISTORY, TOOLS, and SCRATCH sets.
- **caveman/core/compressor.py**: Contains `compress_triples()`, `generate_caveman_prose()`, and `_enforce_caveman_output()` for prose generation.
- **caveman/benchmark/metrics.py**: Core metrics module including `count_tokens()`, `sdpt()`, and `calculate_sdpt()`.
- **caveman/benchmark/run_benchmark.py**: The 8-case QA evaluation suite that performs baseline vs. Charon SDpT comparisons.

## Running the benchmark

To execute the Charon compression benchmark, use the following command from the root directory:

```bash
$env:PYTHONPATH="."; .\.venv\Scripts\python.exe -m caveman.benchmark.run_benchmark
```

Results will be saved to `benchmarks/caveman_benchmark_results.json`.

## Known Limitations

- **Case 8 (photosynthesis)**: Compressed text slightly exceeds raw token count. This is caused by the SLM's tendency toward verbosity when describing simple biological facts, indicating a need for finer prompt calibration.
- **Case 6 failure**: Extreme compression (79.5% reduction) resulted in the loss of critical context. Implementation of a minimum ACU floor is planned for future iterations.
- **L1 FACTS budget**: Currently set to a conservative 100 tokens. Increasing this budget improves retrieval accuracy but negatively impacts the overall compression ratio.
